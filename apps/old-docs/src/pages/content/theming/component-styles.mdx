<section id="section-overview">
## Overview

Every seedui component supports global style customization through the
`theme.components` configuration. You can apply static CSS overrides and
conditional styles that respond to component props and theme values &mdash; all without
touching individual component instances.

<Callout>
  Global component styles are applied via the <code>ThemeProvider</code>'s{' '}
  <code>theme</code> prop. They affect every instance of the component across your app.
</Callout>
</section>

<section id="section-static-style-overrides">
## Static style overrides

Use the `styles` property to apply CSS overrides to every instance of a
component. The value is a standard CSS-in-JS object.

```tsx
import { ThemeProvider, ThemeCustomization } from '@seedui-react/seedui';

const theme: ThemeCustomization = {
  components: {
    button: {
      styles: {
        borderRadius: '999px',
        textTransform: 'uppercase',
        letterSpacing: '0.05em',
      },
    },
    card: {
      styles: {
        border: '2px solid #e2e8f0',
        boxShadow: 'none',
      },
    },
    input: {
      styles: {
        borderRadius: '12px',
      },
    },
  },
};

function App() {
  return (
    <ThemeProvider mode="light" theme={theme}>
      <YourApp />
    </ThemeProvider>
  );
}
```
</section>

<section id="section-conditional-styles">
## Conditional styles

Use `conditionalStyles` to apply styles only when certain conditions are met.
Each entry has a `condition` function that receives the component's props and
the current theme, and a `styles` object to apply when the condition
returns `true`.

```tsx
const theme: ThemeCustomization = {
  components: {
    button: {
      conditionalStyles: [
        {
          condition: (props) => props.disabled === true,
          styles: {
            opacity: 0.4,
            cursor: 'not-allowed',
          },
        },
        {
          condition: (props, theme) =>
            props.color === 'primary' && props.variant === 'filled',
          styles: {
            boxShadow: '0 0 0 3px rgba(99, 102, 241, 0.3)',
          },
        },
      ],
    },
  },
};
```

Conditional styles are evaluated in order. When multiple conditions match, their styles
are merged, with later entries taking precedence.
</section>

<section id="section-combining-static-and-conditional-styles">
## Combining static and conditional styles

You can use both `styles` and `conditionalStyles` together. Static
styles are applied first, then conditional styles are layered on top.

```tsx
const theme: ThemeCustomization = {
  components: {
    tag: {
      styles: {
        fontWeight: 600,
        borderRadius: '999px',
      },
      conditionalStyles: [
        {
          condition: (props) => props.size === 'sm',
          styles: {
            fontSize: '11px',
            padding: '2px 8px',
          },
        },
        {
          condition: (props) => props.size === 'lg',
          styles: {
            fontSize: '15px',
            padding: '8px 16px',
          },
        },
      ],
    },
  },
};
```
</section>

<section id="section-using-the-theme-in-conditions">
## Using the theme in conditions

The `condition` function receives the current theme as its second argument,
allowing you to write mode-aware or token-aware conditional styles.

```tsx
const theme: ThemeCustomization = {
  components: {
    card: {
      conditionalStyles: [
        {
          condition: (_props, theme) => theme.mode === 'dark',
          styles: {
            boxShadow: '0 0 20px rgba(99, 102, 241, 0.15)',
            border: '1px solid rgba(99, 102, 241, 0.2)',
          },
        },
      ],
    },
  },
};
```
</section>
