## Overview

<OverviewDemo example={doc.example} />

Button is the primary interactive element for triggering actions. Use it for form submissions, confirmations, navigation triggers, or any click-based interaction. Choose between filled and transparent variants to establish visual hierarchy — filled for primary actions and transparent for secondary or less prominent ones. Pair the color prop with the context: primary for main actions, neutral for general-purpose buttons, and error for destructive operations like deleting or cancelling.

<section id="section-usage">
## Usage

### Color schemes

Use the color prop to match the button to its intent. Primary for main actions, neutral for general-purpose, and error for destructive operations.

<ComponentPlayground code={doc.usageExamples[0].code} />

### Variants

Filled buttons draw the most attention and are best for primary actions. Transparent buttons work well for secondary or less prominent actions.

<ComponentPlayground code={doc.usageExamples[1].code} />

### Sizes

Three sizes are available. Use small for dense UIs or inline actions, medium as the default, and large for prominent call-to-action buttons.

<ComponentPlayground code={doc.usageExamples[2].code} />

### Loading state

Set isLoading to show a spinner and disable interaction. Useful while waiting for an async operation like a form submission.

<ComponentPlayground code={doc.usageExamples[3].code} />

### Disabled

Disabled buttons are visually muted and ignore clicks. Use them to indicate that an action is unavailable until a condition is met.

<ComponentPlayground code={doc.usageExamples[4].code} />
